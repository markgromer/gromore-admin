CRM Event Messages
Sweep and Go events can now trigger real outbound actions, retries, internal alerts, and recovery handling.
How this works: when a supported Sweep and Go event arrives, Warren queues the matching client message and internal alert actions here. Payment accepted events automatically resolve queued failed-payment follow-ups, and subscription unpaused events resolve queued paused-subscription follow-ups.
Owner alerts use these emails. If this is blank, Warren falls back to internal brand contacts and upgrade contacts.
Supported triggers right now:
client:client_payment_declined
client:invoice_finalized
client:subscription_canceled
client:subscription_paused
client:free_quote_created / quote:created

Recovery events include payment accepted, subscription unpaused, and client/subscription activation events.
