Setup Guide

One-time setup to connect all your brands
How this works: You create one Google app + one Meta app (both free). Enter those credentials in Settings. Then for each brand, click "Connect Google" or "Connect Meta" on the brand detail page. Reports pull data automatically after that.
1 Create a Google OAuth App (covers GA4 + Search Console for all brands)
  1. Go to Google Cloud Console: https://console.cloud.google.com
    Create a new project (or use an existing one). Name it something like "Agency Reports".
  2. Enable these APIs (search for each in the API Library):
    • Google Analytics Data API
    • Google Analytics Admin API
    • Google Search Console API
  3. Configure the OAuth consent screen:
    • User Type: External (unless you have a Google Workspace org)
    • App name: anything (e.g. "My Agency Reports")
    • Support email: your email
    • Scopes: add analytics.readonly and webmasters.readonly
    • Test users: add the Google accounts that own the GA4/GSC properties you want to access
    Important: While the app is in "Testing" mode, only the test users you add can authorize. This is fine for an agency - just add each client's Google account email (or your own if you manage their GA). You do NOT need to publish/verify the app.
  4. Create OAuth credentials:
    • Go to Credentials > Create Credentials > OAuth client ID
    • Application type: Web application
    • Authorized redirect URI: {{ app_url }}/oauth/google/callback
  5. Copy the Client ID and Client Secret, then paste them into Settings > Google OAuth.
2 Create a Meta (Facebook) App (covers Facebook/Instagram Ads for all brands)
  1. Go to Meta for Developers: https://developers.facebook.com
    Click "My Apps" > "Create App".
  2. App type: select "Business".
    App name: anything (e.g. "Agency Ad Reports").
  3. Under your app, go to App Settings > Basic:
    • Copy the App ID and App Secret
    • Add your domain to "App Domains" (e.g. {{ app_url.split('//')[1].split('/')[0] if '//' in app_url else app_url }})
  4. Go to Use Cases > Customize and add Marketing API.
    Under Marketing API permissions, enable:
    • ads_read
    • read_insights
  5. Under Facebook Login > Settings:
    • Valid OAuth Redirect URI: {{ app_url }}/oauth/meta/callback
  6. Paste the App ID and App Secret into Settings > Meta OAuth.
Note: Meta apps in "Development" mode only work for users with a role on the app (Admin/Developer/Tester). Go to App Roles and add yourself. This is fine for agency use - you only need your own account to have access to the ad accounts.
3 Connect Each Brand

Once the OAuth credentials are saved in Settings, connecting brands is simple:

  1. Go to a brand's detail page
  2. Click "Connect" next to Google or Meta
  3. Authorize with the Google/Facebook account that owns the data
  4. For Google: pick the GA4 property and Search Console site from the list
  5. For Meta: pick the ad account
  6. Done. The system will use those connections to pull data when generating reports.
4 Monthly Workflow

After all brands are connected, your monthly process is two clicks:

Generate All Reports
Sidebar > "Generate All Reports"
Pulls data from APIs, runs analytics, scores against benchmarks, builds HTML reports for every brand.
Send All Reports
Sidebar > "Send All Reports"
Emails client-facing reports to every contact marked with auto-send. Only sends unsent reports.
Alternative: CSV Upload (no OAuth setup needed)

If you'd rather not set up OAuth, or for one-off reports, use CSV uploads on each brand's detail page:

  • Google Analytics: GA4 > Reports > Export as CSV
  • Meta Ads: Ads Manager > Export Table Data as CSV
  • Search Console: Performance report > Export as CSV

The parser handles most column name variations from standard GA4, Meta Ads Manager, and GSC exports automatically.
